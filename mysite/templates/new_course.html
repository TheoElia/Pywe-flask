Taking Input

When you write an application, one way or the other, users will have to interact with your application by entering a certain form of data and then your application processes the data and returns a result. How do we allow a user to input data so that we can process?
In Python 2 (there is a slight difference in Python 3, it makes use of input, not raw_input...), taking an input from the user is done using:
raw_input("prompt")
This means that the input the user enters will be considered a string by Python even if he enters numbers.
We can assign a variable to this input by equating a variable name to the code like this:
User = raw_input ("Type anything:") So whatever the user types will now be stored in the variable User. The string in the bracket after raw_input is just the prompt you want to give the user: a little information that tells the user what to type. You can then manipulate what the user types and return information if needed. Since raw_input converts whatever the user types into a string, you can convert it to the data type you want by applying the data type to the user input like this: This is to convert what the user types into an integer
int(User)
We can then assign the result to another variable or the same variable which will mean we have updated the variable.
User = int(User) The same way, you could apply a float too:
User = float(User)
For now we will just greet a user by name. To ask the user to enter his/her name:
Username = raw_input("Enter your name:")
Now let's greet the user:
print ("Hello ",Username)

Functions

The idea of functions from Algebra has been extended in programming. Functions are used as switches to performing actions, the name of the function representing the switch. So you simply call the function and a predefined code is executed. Just like you put on a switch and the light comes on, a function acts as the switch, the inner wirings of the switch can be compared to the definition of the function suite without which both the switch and the function will do nothing. In the English language, functions will be as verbs, which tell the computer to perform a certain action. A verb like polish tell you to polish, but in order to polish, you need to know what to polish, this is what we call in programming a parameter, or an argument. So in "polish the shoe", the function name is polish and the shoe is the input, parameter or argument which helps one know what exactly to polish. You will however agree that, not a every verb requires an input or even a physical output , a verb like "jump" or ‚Äúthink‚Äù can be executed without any further information and may not require any physical output to show that it has been executed. This is what is called void functions in programming, they don't need parameters and might not return anything as well. So, you don't need to always supply a parameter, you only do that if your function requires it. Engineering might view functions as parts an entity that perform certain tasks and together with other functions make the entity work well. Programming is actually engineering in an abstract world, so, when we get to classes we will see how functions come together to help engineer a whole class or entity. You can therefore view functions as verbs, functions from algebra, switches and even parts of an object.
